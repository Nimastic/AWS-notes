\servicesection{Amazon RDS \& Aurora}{\bothtag}{%
  Managed relational databases. RDS for standard engines; Aurora for AWS-native performance.}

\subsection{Amazon RDS}

\begin{keyfacts}
\begin{itemize}
  \item Supported engines: MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, Db2
  \item AWS manages: patching, backups, hardware, failover
  \item \textbf{Multi-AZ} — synchronous standby in another AZ; automatic failover
        (1–2 min); standby is \emph{not} readable (except Multi-AZ cluster mode)
  \item \textbf{Read Replicas} — asynchronous; up to 5 per source; can be cross-region.
        Promote to standalone if needed.
  \item \textbf{Automated backups} — 1–35 day retention; point-in-time restore
  \item Storage: gp2/gp3/io1; auto-scaling storage available
\end{itemize}
\end{keyfacts}

\begin{examtip}
\textbf{Read Replica for DR:} you can promote a cross-region Read Replica to a
standalone DB, making it a valid (though async) DR option. For near-zero RPO/RTO,
use \textbf{Aurora Global Database}.
\end{examtip}

\begin{gotcha}
Encrypting an existing unencrypted RDS instance is \textbf{not} possible directly.
You must: take a snapshot → copy snapshot with encryption → restore from encrypted snapshot.
\end{gotcha}
